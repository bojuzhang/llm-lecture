从序列模型到 Attention

大家好，欢迎来到今天的课堂。今天我们的主题是大语言模型的推理，内容分为三个部分：首先我会带大家回顾 Transformer 的推理流程，然后第二部分介绍推理加速技术，第三部分探讨一个有趣的问题——能不能通过增加推理时的计算量来提升模型效果。

先来看这张对比图。左边是我们熟悉的 RNN，处理序列信息时必须一个一个地传递，信息从 $h_1 \to h_2 \to h_3$，依次串行传递。这就像传话游戏，第一个人的信息要一个个传到最后，中间很容易丢失，训练时还会遇到梯度消失的问题，长距离依赖关系很难学到。

右边是 Transformer 的核心创新——Attention 机制。它的关键在于：每个位置可以直接"看到"所有其他位置，不需要串行传递。就像所有人同时在一个会议室里，谁想说什么直接说，不需要传话。这种并行性是后面推理加速的重要基础。

Attention 具象化：从数据库查询到向量加权

"注意力"可以看做是"全局信息查询"

  • 精确查询：key == "张三" → 判断语句
  • 模糊查询：姓"张" → 部分匹配
  • 最通用：向量内积代替判断语句
核心思想

Query 与 Key 的内积 → 相似度权重
softmax 归一化 → 注意力分布
对 Value 加权求和 → 查询结果

那三个 $\frac{1}{3}$，就是 query 对每个 key 的注意力

其实，"注意力"这个名字取得非常不易于理解。这个机制更准确的说法应该叫"全局信息查询"——做一次 Attention 计算，本质上就是去数据库里做了一次查询。我们用一个具体的例子来理解。

假设有一个以人名为 key、以年龄为 value 的数据库：张三→18，张三→20，李四→22，张伟→19。现在我们的 query 是"所有叫张三的人的年龄平均值"。写程序的话，我们会把字符串"张三"和每个 key 做比较，找出匹配的 value，求平均：$(18+20)/2 = 19$。

但很多时候查询并不那么明确。比如我们想查"所有姓张的人的年龄平均值"，这次不是精确匹配，而是模糊匹配。平均数是 $(18+20+19)/3 = 19$。更一般地，查询可能模糊到无法用简单的判断语句完成。最通用的方法是：把 query 和 key 各建模成向量，用向量内积算相似度，以相似度为权重对 value 做加权和。

把新方法套入刚才的例子。把 key 建模成向量：张三→$[1,2,0]$，李四→$[0,0,2]$，张伟→$[1,4,0]$。假设第一个维度为 1 表示姓张。查询表示为向量 $q=[1,0,0]$。用 $q$ 和每个 key 算内积得到权重：1, 1, 0, 1。接下来用 softmax 归一化，再算 value 的加权和：$[\frac{1}{3}, \frac{1}{3}, 0, \frac{1}{3}] \cdot [18, 20, 22, 19] = 19$。那三个 $\frac{1}{3}$，就是 query 对每个 key 的注意力。

Self-Attention 机制

核心公式：

$\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)V$

$\sqrt{d_k}$：防止数值过大而梯度消失

矩阵形式（$n$ 个 token，行向量）：

$Q = \begin{bmatrix} q_1 \\ q_2 \\ \vdots \\ q_n \end{bmatrix}, \quad K^T = \begin{bmatrix} k_1 \\ k_2 \\ \vdots \\ k_n \end{bmatrix}^{\!T}, \quad V = \begin{bmatrix} v_1 \\ v_2 \\ \vdots \\ v_n \end{bmatrix}$

$QK^T$ 的第 $(i,j)$ 元素即 $q_i \cdot k_j$，衡量第 $i$ 个 token 对第 $j$ 个 token 的关注程度

softmax → 归一化为注意力权重
$\text{weight} \cdot V$ → 加权取出内容

$Q$、$K$、$V$ 由 Embedding 矩阵线性变换得到：

$Q = EW^Q, \quad K = EW^K, \quad V = EW^V$

$E$ 为 Embedding 矩阵，三个 $W$ 为可学习参数；也可来自不同输入，写作 $\text{Attention}(QW^Q, KW^K, VW^V)$

  • $E \in \mathbb{R}^{n \times d_{\text{model}}}$
  • $W^Q, W^K \in \mathbb{R}^{d_{\text{model}} \times d_k}$，$W^V \in \mathbb{R}^{d_{\text{model}} \times d_v}$
  • $Q, K \in \mathbb{R}^{n \times d_k}$，$V \in \mathbb{R}^{n \times d_v}$
  • $QK^T \in \mathbb{R}^{n \times n}$（注意力分数矩阵）
数据库类比

Query：你的搜索关键词
Key：每本书的标签/索引
Value：书的实际内容

接下来我们详细拆解 Attention 的计算过程。请看屏幕上的公式 $\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)V$。

其中 $Q$、$K$、$V$ 并不是直接输入的，而是由 Embedding 矩阵经过线性变换得到的：$Q = EW^Q$，$K = EW^K$，$V = EW^V$，其中 $E$ 是 Embedding 矩阵，三个 $W$ 是可学习的参数矩阵。具体地，$E$ 的大小是 $n \times d_{\text{model}}$，$W^Q$ 和 $W^K$ 的大小是 $d_{\text{model}} \times d_k$，$W^V$ 的大小是 $d_{\text{model}} \times d_v$。当然，$Q$、$K$、$V$ 也可以分别来自不同的 Embedding 输入，这时候就写作 $\text{Attention}(QW^Q, KW^K, VW^V)$。

为了方便理解，我们用一个图书馆的类比。想象你去图书馆找资料，你在搜索框输入的关键词就是 Query——你想了解什么。每本书上贴的标签和索引就是 Key——这本书能提供什么信息。书的实际内容就是 Value。

计算过程是这样的：首先 $Q$ 和 $K$ 做内积，得到一个"相关性分数"——你的关键词和每本书的标签有多匹配。然后过一遍 softmax，把这些分数变成权重，加起来等于 1。最后用这些权重对 $V$ 做加权求和，得到最终输出。

公式里除以 $\sqrt{d_k}$ 这一步很重要：当维度 $d_k$ 比较大时，$QK^T$ 的内积数值会很大，导致 softmax 输出接近 one-hot，梯度几乎为零。除以 $\sqrt{d_k}$ 可以防止数值过大而梯度消失。

Multi-Head Attention

$\text{head}_i = \text{Attention}(QW_i^Q,\; KW_i^K,\; VW_i^V)$
$\text{MultiHead}(Q,K,V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h)\,W^O$

为什么不只用一个 Attention？

  • 每个 Head 学习不同的注意力模式
  • Head 1 可能关注语法关系
  • Head 2 可能关注语义相似性
  • Head 3 可能关注位置关系...
  • $W^O$（$hd_v \!\times\! d_{\text{model}}$）将各头信息线性组合（"融会贯通"），再传给下一层
关键参数

LLaMA-2-7B: 32 heads, $d_{model}=4096$, $d_{head}=128$

理解了单个 Attention 之后，一个自然的问题是：为什么不只用一个？为什么要搞多头？

答案在左边这张图上。每个 Head 有自己独立的 $Q$、$K$、$V$ 权重矩阵，独立计算 Attention，最后把所有头的结果拼接起来，通过输出投影矩阵 $W^O$ 做线性组合，"融会贯通"后再传给下一层。

关键在于：不同的 Head 会学到不同的注意力模式。比如 Head 1 可能关注语法关系，Head 2 关注语义相似性，Head 3 关注位置关系。通过多个"视角"来看同一句话，模型能捕获更丰富的信息。

那这些头是怎么做到"侧重点不同"的呢？在目前绝大多数的大模型（如 GPT-4、LLaMA）训练中，我们根本没有加任何特殊的代码去强迫它们分工。全靠随机初始化和反向传播的残酷淘汰。打个比方：假设你开了一家公司，招了 8 个员工（8 个头），你想让他们合作完成一个极其复杂的项目（比如预测下一个词）。入职第一天（模型初始化），这 8 个人的大脑是完全随机的，每个人瞎乱干。但如果你发现有两个员工做的事情一模一样，天天产出重复的结果，导致整个团队的效率（Loss，损失函数）降不下来，那么在年底考核（反向传播）时，老板（梯度优化器）就会扣他们的奖金，逼着其中一个人去学点新东西。结果：因为每个人初始化的起点不同，在海量数据和反向传播的"逼迫"下，为了让整个团队的总体错误率降到最低，这 8 个头自然而然地走向了分工。

目前大模型领域极其火热的一个子学科——机械可解释性（Mechanistic Interpretability，比如 Anthropic 团队的研究），已经像做解剖手术一样证实了这点。他们发现哪怕不干预，模型在训练中也会自然进化出：Previous Token Head（前一个词头），专门只看左边紧挨着的那个词；Induction Head（归纳头/复读机头），专门寻找前文出现过的固定搭配——这被认为是大模型拥有 In-Context Learning（上下文学习）能力的核心来源。

举一个具体的例子：LLaMA-2-7B 有 32 个头，每个头的维度是 128，$32 \times 128 = 4096$ 正好等于模型的隐藏维度 4096。这个设计在后续理解推理的计算量时会用到。

Transformer 架构全貌

输入侧：inputs 经 Input Embedding 成为词向量序列，再按位置加上 Positional Encoding，作为 Transformer 层的输入。编码器输入输出 Tokens 数量相等。

Masked Attention

每个位置只能看到当前及之前的 token（因果性约束）。实现：将注意力分数矩阵与上三角 $-\infty$ 矩阵相加，未来位置分数变为 $-\infty$，Softmax 后权重为零。

Add & Norm

类似 ResNet 的 $\text{LayerNorm}(F(x) + x)$，残差连接缓解梯度消失，LayerNorm 对每个 token 的特征维度做归一化。

Feed-Forward Network

一个两层全连接网络：$\text{FFN}(x) = \max(0,\, xW_1 + b_1)\,W_2 + b_2$，中间用 ReLU 激活，存储了模型大部分参数。

输出侧：解码器最后一层的输出经 Linear（$d_v \!\to\! |V|$）和带温度的 Softmax，得到 $|V|$ 维概率向量，采样即可输出词表中对应的词。

此为 Attention 原论文架构（用于翻译：原文丢进 $K$、$V$，翻译目标丢进 $Q$）。现代 LLM 均采用 decoder-only 架构，直接用 Masked Attention 学习下一个 token 该生成什么。

现在我们来看 Transformer 的完整架构。请看左边的结构图。

输入侧：inputs 先经过 Input Embedding 成为词向量序列，再按每个 Token 的位置加上对应的 Positional Encoding，作为后续 Transformer 层的输入。编码器的输入和输出 Tokens 数量相等。

中间有几个关键机制。第一是 Masked Attention——每个位置只能看到当前及之前的 token，不能"偷看"后面的，这是因果性约束，也是自回归生成的基础。具体实现上，是将注意力分数矩阵与一个上三角的 $-\infty$ 矩阵相加，上三角位置的分数变为 $-\infty$，经过 Softmax 后权重为零，从而屏蔽未来位置的信息。第二是 Add & Norm，类似 ResNet 的 $\text{LayerNorm}(F(x) + x)$，残差连接把输入直接加到输出上，缓解深层网络的梯度消失问题；LayerNorm 则对每个 token 的特征维度做归一化，稳定训练过程。第三是 Feed-Forward Network，架构图中的前馈网络其实就是一个两层全连接网络，$\text{FFN}(x) = \max(0, xW_1 + b_1)W_2 + b_2$，中间用 ReLU 作为激活函数，它存储了模型大部分的参数。

输出侧：解码器最后一个 Transformer 层的输出经过一次 Linear，将维度从 $d_v$ 变为词表大小 $|V|$，再经过带温度的 Softmax 归一化，得到 $|V|$ 维的概率向量，每个元素对应词表中一个词的预测概率，采样即可输出对应的词。

最后需要说明的是，左边这张图是 Attention 原论文的架构，用于机器翻译任务：原文序列丢进编码器产生 $K$ 和 $V$，翻译目标丢进解码器作为 $Q$。而在现代的大语言模型中，采用的都是 decoder-only 的架构——没有编码器，直接用 Masked Self-Attention 来学习下一个 token 该生成什么。

自回归生成 (Autoregressive Generation)

核心思想：逐 token 生成，每次根据已有内容预测下一个

$$P(y_t \mid y_{\lt t}, x) = \text{softmax}(W \cdot h_t)$$
  • $x$ = 输入 prompt
  • $y_{\lt t}$ = 已生成的 tokens
  • $h_t$ = Decoder 最后一层的隐状态
  • 每次只生成一个 token，追加后继续
  • 遇到 [EOS] 或达到长度上限时停止
自回归生成与 Decoder-Only 的联系

Decoder-only 架构没有编码器，所有信息都通过 Masked Self-Attention 在自身序列上流动。它的训练目标就是"文本接龙"：给定前文，预测下一个 token。自回归生成正是这一训练范式的推理体现——逐 token 生成，每步都在做同一件事：接龙。这也意味着生成 1000 个 token 需要跑 1000 次前向传播。

有了 Transformer Decoder 的结构，我们来看它是怎么生成文字的。

核心思想很简单：逐个 token 生成，每次根据已有的内容预测下一个。请看左边的示意图：第一步输入起始符 BOS，模型预测出"今"字。第二步把"今"追加进去，模型预测"天"。第三步继续，直到遇到结束符或者达到长度上限。

右边是这个过程的数学表达。$h_t$ 是 Decoder 最后一层输出的隐状态，通过一个线性层加 softmax，得到词表上的概率分布。选择概率最大的或者采样的 token 作为下一个输出。

这里要特别强调自回归生成和 decoder-only 架构的联系。Decoder-only 架构没有编码器，所有信息都通过 Masked Self-Attention 在自身序列上流动。它的训练目标本质上就是"文本接龙"——给定前文，预测下一个 token。自回归生成正是这一训练范式的推理体现：每一步都在做同一件事，接龙。这也意味着生成 1000 个 token 需要跑 1000 次前向传播，这就是 LLM 推理慢的根本原因。记住这一点，后面讲加速技术时会用到。

Prefill vs Decode：推理的两个阶段

进一步细化推理过程，我们可以把它分成两个阶段：Prefill 和 Decode。

请看左边，Prefill 阶段。用户的整个 prompt 一次性送入模型，做的是矩阵乘法——一个 $N \times d$ 的输入矩阵乘以 $d \times d$ 的权重矩阵。这是典型的计算密集型操作，GPU 的算力可以充分利用。

再看右边，Decode 阶段。每次只有一个新 token 进来，做的是向量乘法——一个 $1 \times d$ 的向量乘以 $d \times d$ 的权重矩阵。计算量很小，但每次都要从显存里读取整个权重矩阵。所以瓶颈不在算力，而在显存带宽——这就是 Memory-bound。

打个比方：Prefill 像是快递员一次性送一批快递，效率很高。Decode 像是每次只送一个快递但都要跑一趟仓库取货，大部分时间花在路上。

KV Cache：避免重复计算

接下来看推理加速最基本也最重要的技术——KV Cache。

请看左半边，没有 KV Cache 的情况。每次进来一个新的 token，我们需要算出它的 q，然后和之前所有 token 的 K、V 一起做注意力计算。但如果不缓存，每次都要把之前所有 token 的 K 和 V 重新算一遍。$t=1$ 时算 token a 的 $K$ 和 $V$。$t=2$ 时要把 a 和新的 token b 一起重新算。$t=3$ 又要把 a、b、c 全部重算。这样每步的计算量随序列长度线性增长，总复杂度是 $O(n^3)$，完全不可接受。

右边是有了 KV Cache 的做法。把之前算过的 $K$ 和 $V$ 存下来，以后每次进来新 token，只需要算当前 token 的 $q$、$k$、$v$，把新的 $k$、$v$ 加入 cache，然后用新 token 的 $q$ 和 cache 中所有的 $K$、$V$ 做注意力计算即可。复杂度降为 $O(n^2)$。

Trade-off 很明显：KV Cache 节省了大量重复计算，将复杂度从 $O(n^3)$ 降到 $O(n^2)$，但代价是占用额外的显存来存储这些 $K$ 和 $V$ 向量。具体占多少呢？下一张 slide 来量化分析。

KV Cache 到底占多少显存？请看这个公式。$M_{\text{KV}}$ 等于 $2 \times n_{\text{layers}} \times d_{\text{head}} \times n_{\text{heads}} \times S \times \text{bytes}$。其中 $2$ 是因为要同时存 $K$ 和 $V$ 两份。

我们用 LLaMA-2-7B 的具体参数来算。$n_{\text{layers}}=32$，注意力维度 $4096$，FP16 精度下每个元素 $2$ 字节。当序列长度 $S=4096$ 时，代入公式：$2 \times 32 \times 4096 \times 4096 \times 2 = 2\,\text{GB}$。

也就是说，仅仅是为了存储 KV Cache，序列长度 $S=4096$ 时就要 $2\,\text{GB}$ 显存。如果换成 70B 的模型，或者更长的序列，这个数字会更加夸张。

右边的柱状图直观展示了不同序列长度下的显存占用。大家可以看到，长文本场景下 KV Cache 会成为显存瓶颈。这就是为什么后面会有 PagedAttention 等技术来优化 KV Cache 的管理。

总延迟 $T_{\text{total}} = T_{\text{prefill}} + N \times T_{\text{decode}}$，这里 $N$ 是输出 token 的数量。两个关键指标：TTFT 也就是 Time To First Token，$= T_{\text{prefill}}$，代表用户发出 prompt 后等多久才能看到第一个字。TPOT 也就是 Time Per Output Token，$= T_{\text{decode}}$，代表每生成一个 token 要多久。

关键洞察在右边的瀑布图里。当生成 500 或 1000 个 token 时，Decode 阶段占据了绝大部分时间。因为 Decode 是 Memory-bound 的——每步都要从显存读取整个权重矩阵，但只做一次很小的向量乘法，GPU 算力大量闲置。

所以加速推理的重点，在于优化 Decode 阶段的 Memory-bound 问题。具体怎么优化呢？这就是 Part 2 要讲的内容。

推理瓶颈：Memory-bound vs Compute-bound

核心洞察

LLM 推理的瓶颈在 Decode 阶段的内存带宽，不是算力。加速的关键：减少访存量或提高访存效率。

大家好，承接刚才的内容，我们知道 LLM 推理的瓶颈在 Decode 阶段的内存带宽。这一页我们从更底层的视角来分析。

Roofline Model 是分析硬件性能瓶颈的经典模型。核心概念是算术强度，等于计算量除以访存量。算术强度低意味着 GPU 大量时间在等数据，这就是 Memory-bound。算术强度高意味着 GPU 算力跑满了，这就是 Compute-bound。

看右边的 Roofline 图。在算术强度较低的区域，性能被内存带宽限制，曲线是上升的。到了算术强度足够高时，性能被算力限制，曲线变成平台。LLM 的 Decode 阶段恰恰落在左边这个上升区域——每个 token 要读取几十 GB 的模型权重，但只做几 GFLOP 的计算，算术强度很低。

所以加速推理有三个核心方向：减少访存量，可以减少模型大小，比如说量化；减少访存次数，比如 Flash Attention；或者用小模型来帮忙，像是 Speculative Decoding。接下来我们逐一介绍。

Speculative Decoding（投机解码）

首先介绍一种非常巧妙的加速方法——Speculative Decoding，投机解码。

核心想法用一个类比来理解：实习生写草稿，主管来审核。实习生相当于一个小模型，比如 $7\text{B}$，虽然不是那么准但生成速度很快。主管相当于大模型，比如 $70\text{B}$，虽然很准但每次推理很慢。

具体流程分三步，请看图。第一步 Draft，小模型快速生成 $K$ 个 token 作为草稿。第二步 Verify，大模型用一次前向传播同时验证这 $K$ 个 token。第三步输出，从左到右逐个检查，小模型猜对的就接受，猜错的就拒绝并重新采样。

图上的例子：$K=3$，小模型生成了 $t_1$、$t_2$、$t_3$ 三个 token。大模型验证后 $t_1$ 和 $t_2$ 通过了，$t_3$ 被拒绝。最终一次循环就产出了 $2$ 个有效 token，相当于 $2\times$ 加速。

最关键的性质是：最终输出的概率分布和大模型完全一致，不会有任何质量损失。这个等价性有严格的数学证明，下一页我们来看。

Speculative Decoding：为什么结果等价？

目标：证明用 Speculative Decoding 采样的 token 分布与直接从大模型 $p$ 采样完全一致。

接受概率
$$p_{\text{accept}}(x) = \min\!\left(1,\; \frac{p(x)}{q(x)}\right)$$

$p$ = 大模型概率，$q$ = 小模型概率
如果 $p(x) \geq q(x)$：必接受
如果 $p(x) < q(x)$：以 $p(x)/q(x)$ 的概率接受

拒绝后重采样
$$p_{\text{resample}}(x) = \frac{\max(0,\; p(x) - q(x))}{\sum_{x'} \max(0,\; p(x') - q(x'))}$$

只从 $p > q$ 的区域重新采样
保证最终分布为 $p$

最终采样分布：

$$P_{\text{final}}(x) = \underbrace{q(x) \cdot \min\!\left(1, \frac{p(x)}{q(x)}\right)}_{\text{被接受}} + \underbrace{\left(1 - \sum_{x'} q(x') \min\!\left(1, \frac{p(x')}{q(x')}\right)\right)}_{\text{拒绝概率}} \cdot \underbrace{p_{\text{resample}}(x)}_{\text{重采样}} = p(x) \;\checkmark$$
这一页我们证明 Speculative Decoding 的输出分布与大模型完全一致。

左边是接受概率的公式：$p_{\text{accept}}(x) = \min\!\left(1,\; \frac{p(x)}{q(x)}\right)$，其中 $p$ 是大模型的概率，$q$ 是小模型的概率。如果大模型认为这个 token 的概率大于等于小模型的概率，直接接受。如果小模型给出的概率过高，就以 $p(x)/q(x)$ 的概率接受。

右边是被拒绝后的重采样公式：只从大模型概率高于小模型的那些 token 中重新采样，权重是 $p(x) - q(x)$。

下面是完整的推导：最终采样分布等于被接受的部分加上被拒绝后重采样的部分。经过化简，最终结果恰好等于 $p(x)$，也就是大模型本身的分布。这就是为什么 Speculative Decoding 可以保证输出质量不变。

量化的核心思想很简单：用更少的位数来表示模型的权重。

之前提到 Decode 阶段性能瓶颈的原因在于每次生成一个 token 都要从显存中 load 整个权重矩阵。那我们通过量化减小了这个权重矩阵的大小，就能加速推理。

看右边的表格：FP16 时 LLaMA-2-7B 占 $14\,\text{GB}$，量化到 INT8 只要 $7\,\text{GB}$，INT4 只要 $3.5\,\text{GB}$，可以在消费级 GPU 上运行。量化带来的好处是三重的：减少显存占用、加速推理因为减少了访存量、以及省下的显存可以服务更多请求。质量损失通常很小，在 $1\%$ 到 $3\%$ 之间。

GPTQ：逐层最优量化

问题：直接四舍五入会累积误差。GPTQ 的思路：

$$\hat{W}^* = \arg\min_{\hat{W}} \left\| WX - \hat{W}X \right\|_2^2$$
  • 不是单独量化每个权重，而是考虑整层输出误差
  • 利用 Hessian 矩阵 $H = 2XX^T$ 指导量化
  • 逐列量化权重，同时调整未量化的列来补偿误差
  • 只需少量校准数据（~128 条）

简单的四舍五入量化会累积误差，GPTQ (GPT Quantization) 提出了更好的方案。

它的目标是最小化量化后整层输出的误差，公式在左边：寻找一个量化后的权重矩阵，使得它乘以 X 和原始权重乘以 X 尽可能接近。关键技巧是利用 Hessian 矩阵来指导量化——它告诉我们哪些权重对输出影响更大，需要更小心地量化；哪些权重对输出影响不大，量化时可以激进一点。方法是逐列量化，每量化完一列，就调整还没量化的列来补偿误差。

实践中只需要 $128$ 条校准数据，因为你是在量化嘛，不是重新去训一个模型。量化后精度损失极小，不到 $1\%$。

Flash Attention：分块计算

接下来看一个系统层面的优化——Flash Attention。

在计算 softmax 时，要先算出一个 $N \times N$ 的完整注意力矩阵。$N=8192$ 时，这个矩阵有 $6700$ 万元素，单层单头就要 $128\,\text{MB}$ ，而且在计算过程中还会产生一堆中间结果。

计算需要在 SRAM 里进行，但 SRAM 的容量是有限的，无法存下完整的 $N \times N$ 矩阵。传统实现只能把这个矩阵写回 HBM (High Bandwidth Memory) 显存，数据在 SRAM 和 HBM 之间搬来搬去，而搬运数据浪费了很多时间。

Flash Attention 的创新在于分块计算。看右边的示意图：不再是完整的大矩阵，而是把 $Q$、$K$、$V$ 分成小块，在 GPU 的快速 SRAM 缓存里计算局部注意力，然后累积结果。关键是不需要把完整的 $N \times N$ 矩阵写回慢速的 HBM 显存。

数学结果和标准 Attention 完全一致，但显存从 $O(N^2)$ 降到了 $O(N)$。实际效果是在长序列推理时速度提升 $2$ 到 $4$ 倍，显存大幅减少。这是一个非常重要的优化，几乎所有现代 LLM 推理框架都默认使用 Flash Attention。

第一，算法优化：核心就是 Speculative Decoding，用小模型帮大模型做投机执行。

Chain-of-Thought (CoT)

CoT（思维链）— 不直接给答案，而是引导模型逐步推理：把一步复杂映射分解为多步简单映射，用更多计算换更高准确率。
$$P(a \mid q) = \sum_{c} P(a \mid c, q) \cdot P(c \mid q)$$

q=问题， a=答案， c=中间推理步骤  —  引入 c 把一步映射拆成多步

Q: 23 张桌子每张 4 把椅子；14 张坐 3 人、5 张坐 4 人，其余空着，共几人？

✗ 直接回答
82
一步跳到答案，易出错
✓ CoT 分步
1) $14\times3=42$
2) $5\times4=20$
3) $42+20=$ 62 ✓
Circuit Complexity 视角

CoT 等价于增加计算深度；每步 ≈ 网络多一层 → 可表达更复杂的函数。

CoT 是最简单的推理 Scaling 方法：不让模型直接给答案，而是引导它逐步推理，用更多计算换准确率。

数学上，CoT 引入中间步骤 $c$：$P(a \mid q) = \sum_c P(a \mid c, q) \cdot P(c \mid q)$——把原本一步复杂映射拆成多步简单映射。

看左边这张图：上面一行是直接回答，从 $q$ 一步到 $a$，深度 = 1；下面是 CoT，$q \to c_1 \to c_2 \to \cdots \to a$，深度 = $T$。每一步都很简单，总出错率就低。

右上是个餐厅桌椅题做具体示范：直接回答得到 82，是错的；用 CoT 分步：$14 \times 3 = 42$ 人、$5 \times 4 = 20$ 人，加起来 $62$ 人，正确。因为每一步都是个位数乘法或加法，模型不容易出错。

右下是 Circuit Complexity 视角：CoT 本质上增加了神经网络的"计算深度"。理论上有一些问题浅层网络需要指数级宽度才能解决，而深层网络可以高效处理。每个推理步骤就相当于给网络多加了一层。这就是 Inference-Time Compute 的本质——用推理时的深度换表达能力。

CoT 是一条推理路径。Self-Consistency 是它的自然延伸：不只做一次推理，而是采样多次。

请看这张图。同一个问题，我们采样 $N=5$ 次。每次走不同的推理路径，得到不同的答案。路径 1 得到 62，路径 2 得到 62，路径 3 得到 58，路径 4 得到 62，路径 5 得到 65。然后做多数投票：62 有 3 票，58 有 1 票，65 有 1 票。最终选择 62 作为答案。

背后的直觉是：正确的推理路径可能各不相同，但更可能殊途同归，得到同一个正确答案。而错误的路径则是随机分散的，不太可能凑到同一个错误答案上。

计算开销是 $5$ 次推理，等于 $5\times$ 的计算量，但准确率显著提升。这就是 Inference-Time Scaling 的典型例子：花更多计算换取更好的结果。

上一页展示了 Self-Consistency 的原理，这页用量化数据来看投票效果。

以 LLaMA-2-70B 在 GSM8K 数学推理基准上的表现为例，单次推理准确率是 $57\%$。通过多数投票，$N=5$ 时准确率提升到约 $63\%$，$N=11$ 约 $68\%$，$N=21$ 约 $74\%$，$N=41$ 约 $82\%$。

但注意边际收益递减：从 $1$ 次到 $11$ 次提升了 $11$ 个百分点，从 $11$ 次到 $41$ 次只提升 $14$ 个百分点，而计算量增加了近 $3$ 倍。

还有一个重要前提：多数投票要求单次准确率 $p > 0.5$ 才有效。这来自 Condorcet 陪审团定理——如果每个"投票者"正确的概率超过 $1/2$，投票者越多，最终正确的概率就越高。反之如果单次准确率 $p < 0.5$，投票反而会更差。

总的来说，更多采样加投票等于更可靠的答案，但存在收益递减，需要权衡计算成本。

再从结果验证（Best-of-N）进化到过程验证（PRM）。

Best-of-N（结果验证）

  • 生成 N 个候选解法
  • 用 Reward Model 打分
  • 选分数最高的

期望质量随 N 单调增：

$E[\text{quality}] = E\!\left[\max_{i=1}^{N} f(x_i)\right]$

Process Reward Model（过程验证）

  • 不只评估最终答案，评估每一步推理的质量
  • 可以定位具体出错的步骤，引导搜索方向
  • 被推测与 OpenAI o1 的训练思路相关

Best-of-N 用结果验证选优  →  PRM 用过程验证引导搜索：从"投票"到"评分"的进化

Best-of-N 是一种更通用的"多采选优"方法。

看左边的流程图。生成 $5$ 个候选解法，然后交给一个 Verifier 或者 Reward Model 来打分，最后选分数最高的那个。数学上，期望的最优质量会随 $N$ 的增长而增长，当然收益也是递减的。

但更精细的方法是 Process Reward Model，简称 PRM。普通的 Verifier 只看最终答案对不对，而 PRM 评估推理过程中每一步的质量。它可以定位到具体哪一步出了错——比如假设模型在第 3 步算错了乘法，PRM 可以定位到这一步——从而指导搜索算法在正确的方向上探索。

这里特别说明一下：OpenAI 并没有公开 o1 的技术细节，"PRM 是 o1 核心"只是业界推测，因为 OpenAI 在 2023 年发过一篇《Let's Verify Step by Step》——这篇论文在 MATH 数据集上证明了"奖励每一步推理（PRM）"比"只奖励最终答案（ORM）"效果明显更好，所以大家猜 o1 沿用了这条路线。

但 o1 到底用没用 PRM 并不重要，重要的是过程监督这个思路本身——让模型在推理的每一步都能得到反馈、及时纠错，而不是等到最后才发现走错了路。

Tree of Thoughts 把推理从线性 CoT 升级为树搜索。核心思想是：复杂问题常需要试错和回溯，单条推理链一旦出错就无法挽回，而树搜索可以在多条分支中探索和比较。

左边是 ToT 的四个关键组件。第一，思维分解——先定义"一步"是什么粒度，比如 Game of 24 一步是一个等式，创意写作一步是一个段落。第二，思维生成——用 LLM 在每步生成 $b$ 个候选（独立采样或一次性 propose 多个）。第三，状态评估——让 LLM 对中间状态打分 $V(s)$ 或在多个候选间投票。第四，搜索算法——BFS 每层保留 top-$b$，或 DFS 配合阈值剪枝。

ToT 最核心的创新就是：让同一个 LLM 既当生成者，又当评判者——生成者提出想法，评判者评估哪些有前途。这类似人类思考时的"内心对话"。

右边以 Game of 24 为例：用 {1,2,3,8} 算出 24。根节点展开 3 个候选，LLM 给出评分：$8-2=6$ 得 $V=0.8$，$3+1=4$ 得 $V=0.7$，$8+1=9$ 得 $V=0.3$——最后这个直接被剪枝。高分分支继续往下展开，最终找到两条正确路径：$(8-2)\times(3+1)$ 和 $(8+3+1)\times 2$，都等于 24。

看左下的数据：Game of 24 上，直接 CoT 只有 $4\%$，CoT-SC 采样 100 次也只有 $9\%$，而 ToT 配合 $b=5$ 的宽搜直接达到 $74\%$——接近 20 倍的提升。这告诉我们：对需要搜索的问题，树结构比多次独立采样更高效。

蒙特卡洛树搜索 (MCTS)

ToT 的 BFS/DFS 评估噪声大、无探索-利用平衡。MCTS（源自 AlphaGo）通过 多次模拟 降低评估方差，用 UCT 公式 自适应分配搜索预算。
一轮 MCTS 迭代的四个阶段
① 选择 Selection
沿 UCT 最大的子节点下行至叶节点
② 扩展 Expansion
LLM 生成一或多个新 thought 子节点
③ 模拟 Simulation
Rollout / PRM 评分，得到奖励 r
④ 反向传播

UCT — 选点公式
$$\text{UCT}(s,a) = \underbrace{\tfrac{Q(s,a)}{N(s,a)}}_{\text{利用}} + c\cdot\underbrace{\sqrt{\tfrac{\ln N(s)}{N(s,a)}}}_{\text{探索}}$$

$N(s,a)$ 小 ⇒ 探索项大，鼓励冷门分支；
$Q/N$ 高 ⇒ 利用项大，深入已知好路径。

ToT 的 BFS/DFS 有几个不足：BFS 内存开销大、候选平等对待；DFS 容易陷入一条长路径、剪枝阈值需人工设定；最关键是 LLM 单次评估噪声大。MCTS 源自 AlphaGo，正是为解决这些问题而生。

每一轮 MCTS 迭代分四个阶段。第一，选择：从根节点开始沿着 UCT 值最大的子节点一路下行，到达一个尚未完全展开的叶节点。第二，扩展：用 LLM 从该叶节点生成一个或多个新的 thought 作为新子节点。第三，模拟：对新节点进行评估，可以是 rollout 到底拿答案对错，也可以用 PRM 对每步打分。第四，反向传播：把这次模拟的奖励 $r$ 沿路径回传，更新每个节点的访问次数 $N$ 和累积奖励 $Q$。

核心是 UCT 公式。$Q/N$ 是利用项，代表这条路历史平均收益；后面的 $\sqrt{\ln N(s)/N(s,a)}$ 是探索项，访问次数少的节点会获得"好奇心加成"。参数 $c$ 控制探索-利用平衡，经典取 $\sqrt{2}$。Kocsis 和 Szepesvári 2006 年证明了收敛性——模拟次数趋于无穷时 $Q/N$ 收敛到真实价值；实际中 10 到 100 次模拟就能有明显效果。

右边上方的映射表帮助大家建立直觉：MCTS 概念在围棋和 LLM 推理中是一一对应的——状态从棋盘局面变成已有的推理步骤序列；动作从落子变成下一个 thought；策略网络从落子分布变成 LLM 的 token 分布 $p_\theta$；价值网络从胜率估计变成 PRM 分数；终止从胜负变成答案对错。理解了这个映射，就能理解为什么 AlphaGo 的技术可以直接迁移到 LLM 推理。

下方是两个代表性的 MCTS + LLM 系统。第一个是微软亚研院 2025 年初发表的 rStar-Math（arXiv:2501.04519，Guan 等）：用 7B 小模型加 PRM 和 MCTS 做 4 轮自我进化，在 MATH 基准达到 90.0%、AIME 达到 53.3%，首次让小模型在数学推理上超过 o1-preview。第二个是 DeepMind 2024 年 7 月发布的 AlphaProof（详见 DeepMind 博客及 2025 年 Nature 论文）：它把 AlphaZero 的强化学习架构和 Lean 4 形式化证明结合起来，配合 AlphaGeometry 2 在 2024 年 IMO 解出 6 题中的 4 题（AlphaProof 单独解出 P1/P2/P6），拿到银牌水平（28/42 分）。

Inference Scaling Laws

训练时代有 Chinchilla Scaling Laws，推理时代也有类似规律吗？答案是肯定的——性能与推理计算量呈对数线性关系。
$$\text{Performance} \propto \log(\text{Inference FLOPs})$$
  • 推理计算量与性能呈对数线性关系
  • 与训练 scaling law 形式类似（$\text{Loss} \propto \log(\text{Compute})$）
  • 小模型 + 更多推理计算 可以匹配大模型的性能
  • 存在"计算最优"的分配策略
实践意义

不一定需要训练更大的模型。对于单个困难问题，
增加推理计算（更多采样、搜索）可以显著提升准确率。
